## Iteration 4 - TASK-004: Harden web Dockerfile
- Added HEALTHCHECK directive using `wget -q --spider http://127.0.0.1:8080/` (wget available in alpine, curl is not)
- Configured non-root user: nginx runs as uid=101 (nginx user), verified with `docker exec whoami`
- Changed listen port from 80 to 8080 (non-root can't bind privileged ports), updated docker-compose.yml port mapping
- Fixed nginx pid file path to /tmp/nginx.pid (writable by non-root)
- Added deferred upstream resolution via `set $api_upstream` + `resolver 127.0.0.11` to prevent startup crash when api service is unavailable
- Health check verified as "healthy" via `docker inspect --format='{{.State.Health.Status}}'`
- Learnings: Use 127.0.0.1 not localhost in healthchecks (IPv6 localhost resolution fails when nginx only listens on IPv4); nginx pid path in alpine is `/run/nginx.pid` not `/var/run/nginx.pid`; use variable-based proxy_pass to defer upstream DNS resolution
