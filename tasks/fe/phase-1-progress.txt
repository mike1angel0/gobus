- **TASK-008**: AuthProvider with login/register/logout/changePassword/forgotPassword/resetPassword. Refresh token in localStorage, access token in memory. Auto-refresh 60s before JWT expiry. 401 during refresh → full logout; 403/423 during restore → silent clear.

### Auth Pages (Iteration 9)
- **TASK-009**: Login page with React Hook Form + Zod + zodResolver. API error handling for INVALID_CREDENTIALS/ACCOUNT_SUSPENDED/ACCOUNT_LOCKED. Field-level RFC 9457 error mapping. Role-based redirect. Used h1 for heading semantics (CardTitle is div). Shadcn Label manually installed.

### Key Patterns & Gotchas
- **react-refresh/only-export-components**: Never mix component exports with non-component exports (constants, schemas, contexts). Split into separate `.ts` files.
- **shadcn CLI**: Fails with "Invalid configuration" on this project — manually create components instead.
- **ESLint test overrides**: `max-lines-per-function: off` and `max-lines: off` for test files.
- **Complex forms**: Extract field groups into sub-components to stay under complexity/max-lines limits.
- **ApiError constructor**: Takes an `ErrorResponse` object, not individual args.
- **jsdom localStorage**: Must mock with `Object.defineProperty(globalThis, 'localStorage', {...})`.
- **CardTitle**: Renders as `<div>`, not a heading — use `<h1>` directly for accessibility.
- **openapi-fetch middleware**: Must `response.clone().json()` since body stream is single-consume.
- **tailwindcss-animate**: Regular dep, not devDep (imported at build time).
- **Coverage**: Exclude shadcn `use-toast.ts` (~180 lines generated code) alongside `components/ui/`.

## Iteration 10 - TASK-010: Create register page
- Created register-schema.ts with Zod validation matching OpenAPI spec (email max 255, name max 100, password min 8/max 128 with uppercase+lowercase+digit pattern, phone max 20, providerName max 200 conditional on PROVIDER role, confirmPassword match)
- Created register-fields.tsx with extracted RoleToggle, IdentityFields, PasswordFields components to keep main page under complexity limits
- Created register.tsx with role toggle (PASSENGER/PROVIDER), conditional provider fields, password strength indicator (weak/fair/strong with progressbar), API error mapping (EMAIL_ALREADY_EXISTS → email field, field-level RFC 9457 errors), auto-login after success
- 48 tests covering rendering, accessibility, password strength indicator, client-side validation (empty form, invalid email, password rules, confirm mismatch, provider without name), form submission (passenger, provider, phone), API errors (409, 400 field errors, generic), redirect
- Added max-lines: 'off' for test files in ESLint config (test files naturally exceed 500 lines)
- Learnings: For complex forms, extract field groups into separate components to stay under ESLint complexity/max-lines-per-function limits. React Compiler's react-hooks/incompatible-library warning on watch() is expected — React Hook Form's watch API can't be memoized but it's the standard way to observe field values. Use it.each for repetitive schema validation tests to save lines.
