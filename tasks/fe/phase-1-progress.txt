## Iteration 6 - TASK-006: Create typed API client
- Installed openapi-fetch, created src/api/client.ts (typed client with auth middleware), src/api/errors.ts (ApiError class, RFC 9457 parsing), src/api/hooks.ts (useApiClient hook)
- Access token stored in memory only (setAccessToken/getAccessToken), never localStorage
- Auth middleware auto-attaches Bearer token and fires onUnauthorized callback on 401
- Error middleware parses RFC 9457 responses into ApiError instances, falls back to generic error for non-conforming responses
- 28 unit tests covering error parsing, type guards, token management, and hook stability
- Learnings: openapi-fetch middleware uses onRequest/onResponse pattern. The middleware must clone the response before reading the body (response.clone().json()) since the body stream can only be consumed once. Keep token in module-level variable for simplicity — auth context will call setAccessToken().

## Iteration 8 - TASK-008: Create auth context and hooks with secure token handling
- Created src/contexts/auth-types.ts with AuthContext, types (User, RegisterData, AuthStatus, AuthContextValue), JWT helpers, and localStorage helpers
- Created src/contexts/auth-context.tsx with AuthProvider: login, register, logout, changePassword, forgotPassword, resetPassword, auto-refresh, session restore
- Created src/hooks/useAuth.ts hook consuming AuthContext
- Access token stored in memory only (setAccessToken from api/client), refresh token in localStorage
- Auto-refresh schedules timer using JWT exp claim, refreshes 60s before expiry
- On 401 during refresh → full logout; on 403/423 during restore → silent clearAuth
- Wired AuthProvider into main.tsx, updated renderWithProviders with optional withAuth flag
- 15 unit tests covering: initial state, login, register, logout, changePassword, forgotPassword, resetPassword, auto-refresh, error scenarios, useAuth outside provider
- Learnings: jsdom in vitest doesn't provide a proper localStorage by default — must mock it with Object.defineProperty(globalThis, 'localStorage', {...}). Also react-refresh/only-export-components rule forces separating createContext + types into a non-component .ts file and keeping only the component export in the .tsx file (same pattern as TASK-007).
