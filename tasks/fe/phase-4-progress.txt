## Iteration 5 - TASK-005: Create admin hooks
- Created `src/hooks/use-admin.ts` with `useAdminBuses(params)` and `useToggleSeat()` hooks
- `useAdminBuses` supports page, pageSize, and providerId filter params, uses adminKeys.buses for query keys, staleTime 60s

- Dynamic toast messages: "Seat enabled"/"Seat disabled" based on isEnabled value
- 9 tests covering: paginated fetch, providerId filter, default params, error surfacing, disable+toast, enable+toast, ApiError detail, ApiError title fallback, generic error
- Learnings: Admin hooks follow exact same pattern as provider hooks. Invalidate both admin and provider query keys when admin mutates shared resources (seats are visible in both admin and provider views).
