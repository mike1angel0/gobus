### Key Patterns & Gotchas
- Run vitest from `apps/web/`, not monorepo root
- Follow existing page patterns: sub-components for skeleton/error/empty/card, section with sr-only h2
- No Badge UI component — use styled spans
- Avoid setState in effects (use lazy initializer); don't update refs during render (use effects)
- Use `format(new Date(...), 'HH:mm')` in tests for timezone independence
- `offsetMinutes !== ''` not `offsetMinutes ?` — `'0'` is falsy in JS
- Zod: use `required_error` on `z.enum` for custom messages; gives "Required" for undefined numbers, not min error
- `gridcell` role doesn't support `aria-pressed` — use descriptive aria-label
- Extract sub-components to stay under ESLint max complexity (15)
- Wrap useMemo deps on stable references, not derived `??` expressions
- ErrorBoundary in AppLayout (wrapping Outlet) > per-route boundaries
- Use `_error` prefix for unused params to satisfy lint
- When removing inline components, check all imports — other components may share them
- vitest-axe needs explicit `expect.extend(matchers)` in setup; use `import * as axeMatchers from 'vitest-axe/matchers'`
- Admin hooks follow same pattern as provider hooks; invalidate both admin + provider query keys for shared resources
