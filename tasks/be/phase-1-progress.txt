### Key Patterns & Conventions
- API-first: spec/openapi.yaml is source of truth, implement to match
- Response envelope: `{ data: T }` single, `{ data: T[], meta: PaginationMeta }` lists
- Error format: RFC 9457 Problem Details
- Layered architecture: api/ → application/ → domain/ ← infrastructure/
- Lazy singletons for env (getEnv()) and Prisma client
- Reusable $ref parameters (PageParam, PageSizeParam, IdParam) in spec
- Named Prisma relations when multiple FKs point to same model

### Important Gotchas
- OpenAPI 3.1: use `type: ['string', 'null']` not `nullable: true`
- Zod 4: import from "zod" (not "zod/v4")
- Module-level parseEnv() breaks tests — use lazy getEnv() instead
- `vi.stubEnv(key, undefined)` doesn't delete env vars; use `delete process.env.KEY`; add `unstubEnvs: true` to vitest config
- Prisma 7 requires driver adapters; generated client dir must be excluded from tsc and eslint
- Prisma 7 uses `migrations.seed` in prisma.config.ts (not package.json `prisma.seed`); doesn't auto-load .env
- Fastify error handler receives `unknown` type in strict mode — cast to `FastifyError` for `.validation`
- `@fastify/swagger` StaticDocumentSpec expects `OpenAPIV3.Document` type; spec must be pre-bundled
- `__dirname` resolves to src/ at runtime; spec path is 3 levels up
- ESLint 10 uses flat config (.mjs); use `tseslint.config()` wrapper
- Old package-lock.json must be removed when restructuring to workspaces
- `as const` is compile-time only — don't test runtime immutability
- Always check git log before implementing — prior commits may already have done the work

### Useful Commands
- `npm run spec:lint` — validate OpenAPI spec
- `npm run spec:bundle` — bundle to spec/dist/openapi.json
- `npm run spec:preview` — Redoc interactive docs
- `npm run dev` / `npm run build` / `npm run start` — API dev/build/run
- `npm run typecheck` / `npm run lint` / `npm run format` — code quality
- `npx prisma db push` / `npx prisma generate` / `npx prisma db seed` — database
