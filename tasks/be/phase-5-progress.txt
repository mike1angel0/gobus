# Progress Log

## Compacted History
Iterations 1–21 (TASK-001 through US-QA-007, plus fix iteration 17) are summarized below.

### Key Patterns Discovered
- **Prisma mocking**: `vi.fn((fn) => fn(tx))` for `$transaction`; expose `_tx` on mock for test access; `mockResolvedValueOnce` chaining for methods called multiple times with different args
- **Prisma `$queryRaw`**: Tagged template literals auto-parameterize (SQL injection safe); BigInt from COUNT needs `Number()` conversion; mocks must match `$queryRaw` (not `$queryRawUnsafe`)
- **Module-level side-effect testing**: `vi.resetModules()` + `vi.doMock()` to re-import modules with different env; Fastify `app.inject()` cannot add routes after `app.ready()`
- **Soft-delete pattern**: `deletedAt DateTime?` column; auth plugin checks deletedAt → 401; login rejects as invalid credentials; services filter out soft-deleted by default
- **Zod-to-OpenAPI parity**: `z.string().date()` validates YYYY-MM-DD strictly; optional string fields often miss `minLength` constraints; response numeric fields need explicit upper bounds
- **Spec workflow**: `npx redocly bundle` required after spec changes (spec-conformance tests read from `spec/dist/openapi.json`); `spec/dist/` is gitignored; run spec commands from monorepo root
- **Test infrastructure**: Integration tests use `vitest.integration.config.ts`; coverage config (`vitest.coverage.config.ts`) combines unit + integration; `createAuthHeader` accepts optional `{ providerId }`

### Important Gotchas
- `@prisma/adapter-pg` v7.5: `onPoolError`/`onConnectionError` callbacks don't exist in types despite docs
- After adding Prisma schema fields, must run `npx prisma generate` before typecheck
- When adding new DB fields (e.g., `deletedAt`), grep all integration test mocks for responses needing the new field
- Pre-existing test failures can mask new issues — fixing root cause (e.g., missing `$queryRaw` mock) often fixes multiple seemingly unrelated failures
- Docker compose `${VAR?message}` syntax provides clear error messages on missing vars
- Python `re.sub` with `\1` backreference can insert `\x01` byte — use raw strings properly
- Server.ts excluded from coverage in vitest config
- `npm run spec:lint` and `npm run spec:bundle` must be run from monorepo root

### Useful Commands
- `npx prisma generate` — regenerate Prisma client after schema changes
- `npx prisma migrate dev --name <name>` — create and apply migration
- `npx redocly bundle spec/openapi.yaml -o spec/dist/openapi.json` — bundle spec
- `npm run spec:lint` — validate OpenAPI spec (from monorepo root)
- `npm run api:validate` — run 66 spec conformance tests
