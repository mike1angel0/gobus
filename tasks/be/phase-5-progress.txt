## Iteration 3 - TASK-003: Fix search service in-memory pagination
- Replaced in-memory filtering + pagination with DB-level approach: raw SQL self-join on StopTime for origin/dest ordering check, separate count query without includes, LIMIT/OFFSET for pagination
- Only fetches full schedule data (with includes) for the paginated subset of IDs
- Learnings: Prisma `$queryRaw` with tagged template literals auto-parameterizes values (safe from SQL injection); BigInt returned by COUNT needs `Number()` conversion; mock `$queryRaw` as `vi.fn()` — Prisma calls it with a Sql object from the tagged template; integration tests use separate vitest config (`vitest.integration.config.ts`)
