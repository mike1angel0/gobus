- Auth plugin checks `deletedAt` and returns 401 (same as "not found" to prevent enumeration)
- Auth service login rejects soft-deleted users as invalid credentials
- Admin service: new `softDeleteUser` method, `listUsers` excludes soft-deleted by default, `updateUserStatus` rejects soft-deleted
- Driver service: `listByProvider` and `delete` exclude soft-deleted drivers
- Schedule service: `validateDriverOwnership` excludes soft-deleted drivers
- New DELETE /api/v1/admin/users/:id endpoint with audit logging
- OpenAPI spec + Zod schemas updated with `deletedAt` field
- Learnings: After adding a Prisma schema field, must run `npx prisma generate` to update the generated client types before typecheck; existing tests that assert exact select/where params need updating when adding new fields to queries; soft-delete is simpler than archive-and-cascade since it requires no schema restructuring
