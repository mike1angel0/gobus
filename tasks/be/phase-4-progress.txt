### Key Patterns
- RFC 9457 Problem Details for all error responses (type, title, status, detail, code, errors[])
- Response envelope: `{ data }` for single, `{ data, meta }` for paginated lists
- Pagination: page/pageSize query params, service returns `{ items, total }`, route maps to `{ data, meta: { page, pageSize, totalItems, totalPages } }`
- Integration tests use `app.inject()` with full plugin stack; auth mocks chain `mockResolvedValueOnce` for auth lookup + service lookups
- Admin endpoints: `preHandler: [app.authenticate, requireAdmin]`
- Audit logging is fire-and-forget with try/catch wrapper (not just .catch()) to handle synchronous errors

### Important Gotchas
- @fastify/rate-limit v10 throws errors that hit the error handler — must handle statusCode 429 explicitly
- Fastify bodyLimit errors arrive as statusCode 413 in error handler
- Prisma JSON fields need `& object` intersection: `(metadata as Record<string, unknown> & object)`
- OpenAPI 3.1 uses `type: ['string', 'null']` not `nullable: true`
- Zod `z.record()` requires 2 args: `z.record(z.string(), z.unknown())`
- `decorateRequest('audit', null)` needs `as unknown as FastifyRequest['audit']` cast
- Booking create uses interactive $transaction (callback), not array-style
- `vi.hoisted()` needed for spies used inside `vi.mock()` factory functions
- Pino `level` must be a valid string (not undefined) — env schema must provide default
- Integration test mocks for suspend path need 3 chained mockResolvedValueOnce (auth + status + revoke)
- When adding pagination, integration test mock must include `.count()` alongside `.findMany()`
- CWD matters for spec:lint — must run from monorepo root

### Useful Commands

- `npm run jobs:cleanup-tokens` — run token cleanup job
