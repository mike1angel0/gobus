### Key Patterns
- RFC 9457 Problem Details for all error responses (type, title, status, detail, code, errors[])
- Response envelope: `{ data }` for single, `{ data, meta }` for paginated lists
- Pagination: page/pageSize query params, service returns `{ items, total }`, route maps to `{ data, meta: { page, pageSize, totalItems, totalPages } }`
- Integration tests use `app.inject()` with full plugin stack; auth mocks chain `mockResolvedValueOnce` for auth lookup + service lookups
- Admin endpoints: `preHandler: [app.authenticate, requireAdmin]`
- Audit logging is fire-and-forget with try/catch wrapper (not just .catch()) to handle synchronous errors
- Use `asserts resource is T` return type on ownership helper so TypeScript narrows automatically after the call
- Centralize token creation in a shared test helper (`createAuthHeader`) to minimize test update blast radius
- When adding `select` to queries, all unit tests asserting `toHaveBeenCalledWith` must include the select clause
- Health/metrics routes need proper Prisma mocks in integration tests

- Fastify bodyLimit errors arrive as statusCode 413 in error handler
- Prisma JSON fields need `& object` intersection: `(metadata as Record<string, unknown> & object)`
- OpenAPI 3.1 uses `type: ['string', 'null']` not `nullable: true`
- Zod `z.record()` requires 2 args: `z.record(z.string(), z.unknown())`
- `decorateRequest('audit', null)` needs `as unknown as FastifyRequest['audit']` cast
- Booking create uses interactive $transaction (callback), not array-style

- Integration test mocks for suspend path need 3 chained mockResolvedValueOnce (auth + status + revoke)
- When adding pagination, integration test mock must include `.count()` alongside `.findMany()`
- CWD matters for spec:lint — must run from monorepo root
- `import.meta.url` fails tsc with CommonJS output — use `process.cwd()` instead
- Driver trips detail uses `date` query param (not `tripDate`)
- Admin suspend needs 3 chained mockUserFindUnique (auth + service findUnique + revokeAllSessions findUnique)
- User `preferences: null` fails spec validation since spec expects object type — mock with `{}`

- Provider integration test mocks must match `select: { provider: true }` pattern
- Monorepo workspace installs need --workspace and --include-workspace-root flags
- Prisma client must be generated in both build and runtime stages (runtime needs the native binary)
- Provider.contactEmail is nullable and not unique — can't use Prisma upsert, use findFirst+create
- striptags v3 is CommonJS but works with ESM import
- `@fastify/helmet` v13 (helmet v8) does not support `permissionsPolicy` — add via onSend hook
- When adding issuer/audience to jwt.verify, ALL test token signing must be updated
