## Iteration 9 - TASK-009: Create driver service
- Created DriverEntity and CreateDriverData types in domain/drivers/driver.entity.ts
- Created DriverService with listByProvider, create, delete methods
- Added assignedScheduleCount to OpenAPI Driver schema (was missing from spec, required by PRD)
- listByProvider uses Prisma _count for driverSchedules to get assigned schedule count
- create hashes password with bcryptjs (12 rounds), sets role=DRIVER, links providerId
- create checks email uniqueness (409 AUTH_EMAIL_TAKEN)
- delete unassigns driver from all schedules first (updateMany driverId=null), then deletes user
- delete also checks role=DRIVER (prevents deleting non-driver users)
- 10 unit tests: list pagination/empty/page2, create happy/phone-null/conflict, delete happy/not-found/ownership/wrong-role
- Patterns: drivers are User records with role=DRIVER, not a separate model. Use prisma.user with where: { role: 'DRIVER' }
